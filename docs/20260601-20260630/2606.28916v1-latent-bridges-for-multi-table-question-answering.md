## 摘要
我们提出了GRAB，一个用于表格问答的构造器-编码器-桥接流水线。我们的方法将关系数据提升为异构图，通过消息传递进行编码，并通过一小组查询条件化的潜在标记将信号传递给大语言模型（LLM）。这为LLM提供了紧凑的、任务相关的结构表示以及扁平化的文本。至关重要的是，LLM严格保持冻结以保留其通用推理能力；我们仅训练轻量级图编码器和潜在桥接（91M参数），使得整个流水线可以高效训练。我们的流水线显著提高了关系问答的性能，在要求高的多表设置中增益最大，提供了一种高效、有原则的方法来连接关系深度学习与LLM。

## Abstract
We introduce GRAB, a constructor-encoder-bridge pipeline for table question answering. Our method lifts relational data into an heterogeneous graph, encodes it via message passing, and transfers the signals to an LLM through a small set of query-conditioned latent tokens. This provides the LLM with a compact, task-relevant structural representation together with the flattened text. Crucially, the LLM remains strictly frozen to preserve its general reasoning capabilities; we train only the lightweight graph encoder and latent bridge (91M parameters), allowing the entire pipeline to be trained efficiently. Our pipeline significantly improves performance on relational Question Answering, with the largest gains in demanding multi-table settings, offering an efficient, principled way to connect relational deep learning with LLMs.

## 1. 论文的核心问题与整体含义

- **核心问题**：多表关系问答（Multi-Table QA）中，表格并非孤立的扁平文本，而具有严格的行列结构、外键连接以及跨表依赖关系。现有大语言模型（LLM）通常直接将表格序列化为文本，这会导致结构信息丢失、上下文碎片化，使LLM在多表推理上变得脆弱。
- **整体含义**：论文提出GRAB（Graph-Relational Attention Bridge），以一种**构造器‑编码器‑桥接**流水线，将关系数据显式建模为异构图，并通过查询感知的潜在标记将结构信息注入冻结的LLM。从而**在不牺牲 LLM 通用推理能力的前提下，高效捕捉跨表结构，实现多表问答性能的显著提升**。

## 2. 方法论

### 2.1 核心思想
- 将多表数据（含外键）转换为**三元异构图**（行节点、列类节点、值组节点），使重复值、连接关系等成为图上的显式连接。
- 用**消息传递图神经网络（GNN）**对图编码，提取结构依赖。
- 通过**查询条件化的潜在桥接**将变长的图表示压缩为一组固定长度的**软标记（latent tokens）**，与扁平化文本共同输入冻结的LLM进行答案生成。
- 训练时**严格冻结 LLM**，仅更新轻量级的图编码器与桥接模块（共约 91M 参数）。

### 2.2 关键技术细节
- **图构造器**（constructor γ）：
  - 行节点：每行一个节点，特征为该行“列名:值”拼接的文本嵌入。
  - 列类节点：外键关联的列被合并为同一个节点，特征为列头文本嵌入的平均。
  - 值组节点：对类别/文本值规范化后分组；数值则按分位桶离散化。这些节点主要作为结构性锚点。
  - 边：行‑值边记录行中包含哪些值组；列‑值边记录列包含哪些值组。
- **图编码器**：
  - 采用 L 层消息传递，行与列节点分别从连接的值节点聚合信息，值节点聚合行和列的信息。
  - 更新使用类型特定的残差块（归一化 + 前馈网络），公式大致为：
    \[
    M^{\ell}_V = A^\top_{RV}H^{\ell}_R + A^\top_{CV}H^{\ell}_C, \quad 
    M^{\ell}_R = A_{RV}H^{\ell}_V, \quad 
    M^{\ell}_C = A_{CV}H^{\ell}_V
    \]
    然后依次进行消息融入与 FFN 残差更新。
- **潜在桥接**：
  - 用类似 Perceiver Resampler 的结构，先获取问题嵌入 \(H_Q\)。
  - 初始化可学习查询向量 \(Z^{(0)}\)（按行、列、值分区），分别与对应图节点表示和问题嵌入拼接后进行交叉注意力，生成条件化的潜在标记 \(Z\)。
  - 将 \(Z\) 通过线性投影映射到 LLM 隐空间，作为软前缀注入提示中。
- **训练目标**：只训练图编码器+桥接的参数 \(\Phi\)，最小化生成答案的负对数似然。

## 3. 实验设计

### 3.1 数据集与场景
- **单表问答**（6个）：StructQA、HiTab、WTQ、WikiSQL、HCTQA、TabMWP，覆盖结构敏感、层级表头、复杂布局、数学推理等。
- **多表/混合问答**（7个）：MultiHiertt、SciTaT、MMQA、TQA-Bench、Atis、GeoQuery、Spider，覆盖跨表推理、科学证据、表格输出等。
- 额外任务：TabFact（事实核查）与 Spider（Text‑to‑SQL）用于测试迁移能力。

### 3.2 对比基准
- **推理‑零样本**：仅扁平化表格输入。
- **冻结 LLM 系列**：Prompt Tuning、TAMO（超图表征）。
- **微调 LLM 系列**：LoRA（仅序列化）、GRAB+LoRA。
- **专用模型**：TableLlama（单表）、MultiTabQA（多表）、GPT‑5.4‑mini（闭源大模型）。

### 3.3 评价指标
- 多数数据集采用 **答案集完全匹配准确率**（经标准化后多集合等值要求）；HCTQA 额外报告 F1 与 Complete Containment；TQA‑Bench 为多项选择准确率；SciTaT 分短答案 EM 与自由形式答案 F1；SQL 类（ATIS 等）用表级 EM 与单元级 F1。

- 实验在 **NVIDIA A100 GPU** 上进行。
- **LLM 主干采用 Qwen3‑4B‑Base 并保持冻结**；图编码器及桥接总参数量约 **91M**，宣称可在 **单张 GPU** 上高效训练。

## 5. 实验数量与充分性
- **主实验覆盖 13 个基准**，包括单表、多表、事实核查等，与多种强基线对比，实验规模大。
- **消融实验**：
  - 潜变量数量 K（1‑256）与 GNN 深度 L（1‑16）的扫描；
  - 投影头类型（线性/MLP/深层 MLP）；
  - 去除问题条件化、去除序列化表格；
  - 不同随机种子（5 次）验证稳定性。
- **压力测试**：设计 15 张关系表，独立变化结构轴（0‑3 个过滤条件 + GROUP‑BY 键）与计算轴（LOOKUP/COUNT/MAX/AVG），**系统性地分离结构性瓶颈与算术瓶颈**。
- **对比多模型规模**：使用 Qwen3‑14B 重复压力测试，验证增益并非由参数规模吸收。
- 实验设计**客观且公平**：统一训练配置（优化器、调度、早期停止）、统一的 8192 token 过滤，基线均在相同条件下复现或使用已发布代码。

## 6. 主要结论与发现
- **性能大幅提升**：GRAB 在几乎所有基准上优于序列化及软提示基线，多表、复杂布局场景增益尤为突出（如 HCTQA F1 提高 17.5，Spider 单元 F1 提高 13.3）。
- **冻结 LLM 的 GRAB 与微调 LLM 持平甚至更优**：在 StructQA 上 84.80 vs LoRA 的 77.93，说明**显式结构编码比从文本中隐式恢复结构更高效**。
- **查询条件化优于静态超图**：GRAB 在所有单表基准上超过 TAMO，证明动态分配潜变量容量的重要性。
- **轻量模块可媲美大模型**：4B 参数的流水线在多个基准上超过参数量约 100 倍的 GPT‑5.4‑mini，显示出关系编码的巨大价值。
- **结构定位 vs. 精确算术**：压力测试表明 GRAB 极大减轻了证据定位的负担（LOOKUP、COUNT 大幅增益），但**精确算术（AVG）仍是固有天花板**，因为图编码使用分位数桶而非原始数值。

## 7. 优点
- **LLM 冻结**：保留了预训练模型的通用推理能力，避免灾难性遗忘。
- **轻量化**：仅 91M 可训练参数，单 GPU 可训，降低了使用门槛。
- **结构显式建模**：通过图构造器将重复值、外键转换为图连通性，使消息传递能自然地传播跨行、跨表信息。
- **问题条件化桥接**：使有限的潜变量容量聚焦于当前问题相关的图结构，理论上也得到“草图复杂度分离”的支持。
- **全面的诊断性评估**：提出的“结构深度 × 计算路径”压力测试具有独到性，能精确归因失败来源。
- **稳健性验证**：多种子实验、不同规模模型的压力测试重演，结论可靠。

## 8. 不足与局限
- **预处理开销**：图构建与编码可能在大规模数据库或高度连接的模式下成为性能瓶颈。
- **上下文窗口限制**：由于仍保留序列化文本，模型仍受制于 LLM 的最大上下文长度，超长表格可能被截断。
- **模式敏感性**：如果数据库元数据缺失、有噪声或外键信息不准确，图构造的质量会下降，进而削弱结构化增益。
- **仅支持已检索的表格子集**：假设相关表格已被检索，当面对整个数据库时，需要额外的检索模块。
- **精确算术的限制**：数值被离散化，模型无法在图侧完成精确求和/平均，这部分仍依赖 LLM 对原始文本的算术处理，而 LLM 在此类任务上表现不可靠。
- **未与 SQL 执行器等符号工具深度整合**，回答需要复杂聚合或多步推导时仍可能失败。
